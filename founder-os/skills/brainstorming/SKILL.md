# Brainstorming Skill

A Socratic design refinement process to turn rough ideas into fully formed designs before coding. Based on the Superpowers methodology.

## When to Activate

- Before ANY creative work (features, components, behavior changes).
- When a project or task is first started.
- Even for "simple" tasks to avoid unexamined assumptions.

## The <HARD-GATE>

Do NOT write any code, scaffold any project, or take any implementation action until you have presented a design and the user has explicitly approved it.

## Instructions for Agent

1. **Context First**: Explore existing files, docs, and recent commits.
2. **One Question at a Time**: Ask clarifying questions one by one to understand purpose and constraints.
3. **Propose Approaches**: Offer 2-3 distinct approaches with trade-offs and a recommendation.
4. **Sectioned Design**: Present the design in bite-sized sections. Get approval after each section.
5. **Formal Spec**: Save the final design to `docs/specs/YYYY-MM-DD-{topic}-design.md`.
6. **Self-Review**: Check the spec for placeholders, contradictions, or ambiguity.
7. **Approval**: Ask the founder to review the written spec file before invoking `PATH` planning.

## Transition

Once the spec is approved, transition to the `PATH` (writing-plans) phase.
