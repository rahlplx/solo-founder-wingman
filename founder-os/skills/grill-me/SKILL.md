# Grill Me Skill

A relentless interview session to stress-test a plan or design before implementation begins. Derived from Matt Pocock's alignment methodology.

## When to Activate

- When a founder presents a new idea or plan.
- Before moving from BRIEF (research) to PATH (planning).
- When a task feels ambiguous or under-specified.
- If the founder uses the word "grill" or "stress-test".

## Why Grilling?

The most common failure mode is misalignment. Grilling closes the communication gap by forcing the agent and founder to align on every branch of the decision tree.

## Instructions for Agent

1. **Relentless Interview**: Ask questions that walk down each branch of the design tree.
2. **One at a Time**: Ask exactly ONE question at a time. Batching questions is bewildering for a founder.
3. **Recommend Answers**: For every question you ask, provide your current best recommendation (the "why").
4. **Codebase First**: If a question can be answered by searching the code, do that first instead of asking.
5. **Shared Language**: Use the project's domain jargon (from `CONTEXT.md` or `PRD.md`) to keep the conversation concise.

## Success Criteria

The "grill" is complete when:
- Every ambiguity in the user's request is resolved.
- Both agent and founder have a shared understanding of the "Happy Path" and "Shadow Paths".
- The next step (PATH or SHIP) is crystal clear.
