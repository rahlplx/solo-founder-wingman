# FEATURE-001: The "Freelancer Loop" (Client-Facing Verification & Delivery)

## [Real-World Scenario]
"I'm a freelance designer using the agent to build a landing page for a client. The agent says it's 'done,' and it passed all the internal PATH tests. But I can't just send the client a link to my local dev server or a GitHub repo. I need a way to 'freeze' the work, generate a preview, and get my client's sign-off before I consider the invoice 'billable'."

## [Human-Readable Summary]
The current "verification engine" (PATH/SHIP) is built for the developer. It's too technical for a client. We need a "Freelancer Loop"—a way to package the agent's output into a "Reviewable Deliverable" that a non-technical client can approve.

## [AI-Tailored Technical Requirements]

### 1. New Skill: /share-review
- **Purpose**: Package current work for external stakeholder review.
- **Inputs**: Deliverable type (Web, PDF, Code), Client Contact, Context.
- **Output**:
  - A temporary staging URL (e.g., via Vercel/Netlify).
  - A "Review Summary" in plain English (generated by `code-critic` but in a client-friendly tone).

### 2. New Skill: /process-feedback
- **Purpose**: Ingest client comments and map them back to technical tasks.
- **Logic**: Use the BRIEF framework to convert "I don't like the blue" into a "WHEN/THEN" behavior rule change in `PRD.md`.

### 3. Integrated "Milestone" Tracking
- **Requirement**: Update `PRD.md` to support `Milestones`.
- **Feature**: Tie the `/verify-path` success to a specific Milestone completion, creating a "Proof of Work" log for invoicing.

## [Verification Engine]
- [ ] Create `founder-os/skills/share-review/SKILL.md`.
- [ ] Add `milestones` array to `templates/PRD.md.tpl`.
- [ ] Mock a `/process-feedback` session where a "vague" client comment is successfully turned into a non-goal or a behavior rule.
